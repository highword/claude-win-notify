2. Define public function `pub fn handle_hook() -> Result<(), crate::error::AppError>`:
   - Read all of stdin to a String via `std::io::read_to_string(std::io::stdin())`
   - If stdin is empty, log "Empty stdin received" and return Ok(()) (silent exit)
   - Deserialize JSON with `serde_json::from_str::<HookInput>(&input)`
   - On parse error: call `crate::log::log_error(&format!("JSON parse error: {}", e))` and return Ok(()) (per D-06: exit 0 on failure)
   - Match on `hook_input.hook_event_name.as_str()`:
     - `"Stop"` → call `handle_stop(&hook_input)`
     - `"Notification"` → call `handle_notification(&hook_input)`
     - _ (any other event) → return Ok(()) silently (per D-05)

3. Define `fn handle_stop(input: &HookInput) -> Result<(), crate::error::AppError>`:
   - If `input.stop_hook_active == Some(true)`, return Ok(()) (prevent infinite loop)
   - Extract project name: `std::path::Path::new(&input.cwd).file_name().unwrap_or_default().to_string_lossy().to_string()`
   - Call `crate::toast::show_toast("Claude Code", "\u{2714} Task Complete", &project_name)?`
   - Return Ok(())

4. Define `fn handle_notification(input: &HookInput) -> Result<(), crate::error::AppError>`:
   - Extract project name same as above
   - Extract body: use `input.message.as_deref().unwrap_or("New notification")`
   - Call `crate::toast::show_toast("Claude Code", body, &project_name)?`
   - Return Ok(())

fn escape_xml(s: &str) -> String {
    s.replace('&', "&amp;")
     .replace('<', "&lt;")
     .replace('>', "&gt;")
     .replace('"', "&quot;")
     .replace('\'', "&apos;")
}

3. Define public function `pub fn show_toast(title: &str, body: &str, attribution: &str) -> Result<(), crate::error::AppError>`:
```rust
use windows::core::HSTRING;
use windows::Data::Xml::Dom::XmlDocument;
use windows::UI::Notifications::{ToastNotification, ToastNotificationManager};

pub fn show_toast(title: &str, body: &str, attribution: &str) -> Result<(), crate::error::AppError> {
    let xml_string = format!(
        r#"<toast>
  <visual>
    <binding template="ToastGeneric">
      <text>{}</text>
      <text>{}</text>
      <text placement="attribution">{}</text>
    </binding>
  </visual>
  <audio src="ms-winsoundevent:Notification.Default"/>
</toast>"#,
        escape_xml(title),
        escape_xml(body),
        escape_xml(attribution)
    );

    let toast_xml = XmlDocument::new()?;
    toast_xml.LoadXml(&HSTRING::from(&*xml_string))?;
    let toast = ToastNotification::CreateToastNotification(&toast_xml)?;
    let notifier = ToastNotificationManager::CreateToastNotifierWithId(&HSTRING::from(AUMID))?;
    notifier.Show(&toast)?;
    Ok(())
}
```

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | T (Tampering) | src/toast.rs | mitigate | escape_xml() applied to all user-derived strings before embedding in Toast XML template |
| T-02-04 | D (DoS) | src/hook.rs | mitigate | stdin read with no timeout — acceptable because Claude Code hook system always provides finite input. If manually invoked, TTY detection (Plan 01) prevents hang. |
| T-02-05 | I (Info Disclosure) | src/hook.rs | accept | session_id and paths are displayed in Toast — this is intentional UX, runs on user's own machine |
| T-02-06 | S (Spoofing) | stdin | accept | Claude Code is the intended caller — no authentication needed for a local CLI tool |
